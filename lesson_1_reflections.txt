Why do you think some version control systems, like Git, allow saving multiple
files in one commit, while others, like Google Docs, treat each file separately?

	Pelo que eu entendi, o Git considera arquivos salvos no mesmo repositório
	como se tivessem uma relação entre si. Isso significa que quando o usuário
	fizer uma mudança em um, que tenha repercussão nos outros, eles podem ser
	modificados pelo mesmo commit, sem ter q modificar um por um.

How can you use the commands git log and git diff to view the history of files?

	GIT LOG: Usado para buscar todas os commits de um arquivo. Basta ir na 
	pasta do arquivo em questão e digitar git log. Irá aparecer cada commit,
	com seu respectivo autor, data da mudança e ID, além de uma mensagem escrita
	pelo usuário explicando a mudança realizada. Escrever git log --stat dá 
	as estatísticas relacionadas com as mudanças feitas. Para parar, digite 
	q, que significa quit.

	GIT DIFF: Usado para comparar diferentes versões, commits, de um arquivo. 
	Basta ir na pasta do arquivo em questão e digitar git diff ID_1 ID_2. As 
	linhas em preto não foram modificadas, as linhas em vermelho foram removidas
	e as linhas em verde foram adicionadas. Pode escrever apenas os primeiros 4
	dígitos da ID.
	OBS: Similar ao comando FC, no Windows. Utilizado para comparar dois arquivos
	diferentes. Para configurar as cores de output, basta digitar git config --global color.ui auto.
